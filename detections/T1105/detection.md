# T1105 - Ingress Tool Transfer

## Summary

Ingress Tool Transfer is a technique used by adversaries to transfer tools, scripts, or malware from an external system to a compromised host after gaining initial access.

For this detection, Atomic Red Team was used to simulate file download activity on a Windows endpoint. The resulting telemetry was analyzed in Security Onion and a Sigma rule was developed to identify the behavior.

## Attack Simulation

Atomic Red Team technique T1105 was executed on the Windows endpoint to simulate the download of a file from a remote source.

The test leveraged native Windows utilities commonly used to retrieve files over HTTP or HTTPS. Similar techniques are frequently observed during post-exploitation activity when attackers download additional tooling onto a compromised system.

## Detection

The detection monitors process creation events associated with native Windows utilities capable of downloading content from remote locations.

Detection Logic:

* PowerShell executing `Invoke-WebRequest`
* `certutil.exe` downloading remote content
* `bitsadmin.exe` transferring files
* `curl.exe` retrieving content from external sources

These utilities are legitimate administrative tools but are also commonly abused by attackers to transfer payloads and supporting tools.

## Validation Results

The Atomic Red Team simulation successfully generated process creation telemetry on the endpoint.

Security Onion ingested the resulting events and the Sigma rule generated an alert when download activity was observed.

The generated telemetry provided visibility into:

* Executing User
* Parent Process
* Command Line Arguments
* Download Utility Used
* Remote Resource Referenced

## Hunt Query

```kql
event_data.process.name:(powershell.exe OR certutil.exe OR bitsadmin.exe OR curl.exe)
AND process.command_line:(*http://* OR *https://*)
```

This query was used to identify file download activity generated by native Windows utilities commonly abused for ingress tool transfer. The hunt query successfully identified the Atomic Red Team simulation and validated telemetry visibility within Security Onion.

## Key Takeaways

Ingress Tool Transfer is commonly observed during the early stages of post-exploitation activity. Attackers frequently leverage native Windows utilities to download additional payloads while blending in with legitimate administrative activity.

Monitoring command-line arguments and download-related utilities can provide visibility into suspicious file transfer behavior and help identify adversary activity before malware execution occurs.
